01 — The algorithms

Two real models doing the triage.

🏷️

Auto-tagging

A TF-IDF vectorizer feeds a one-vs-rest Logistic Regression — one classifier per label (crash · ui · performance · auth · database · network). Multi-label, with a probability threshold, trained on a seed set.

🎯

Assignee suggestion

Builds one TF-IDF document per person from the bugs they've resolved, then ranks them by cosine similarity to the new bug. The closest match is suggested.

🔍

Explainable

Both are transparent: the tagger surfaces the terms that drove each label, and the suggester returns the full similarity scoreboard — no black box.

02 — Features

API, CLI, and a tested core.

⚡

FastAPI service

Create / list / get / update bugs, change status, and comment — auto-tag and suggest endpoints too. Interactive docs at /docs.

⌨️

CLI

The same database and algorithms from the terminal: add, list, show, status, tag, suggest.

🗄️

SQLite-backed

A dependency-free data layer over sqlite3: bugs + comments, status lifecycle, label and assignee filters.

🌱

Seeded history

Ships with 24 resolved bugs across 6 people, so assignee suggestions work the moment you start the server.

🔁

Status lifecycle

Every bug moves through open → in-progress → closed, with validation on every transition.

🧪

pytest suite

33 tests over the TestClient: auto-tags, assignee routing, CRUD, status transitions, and comments.
